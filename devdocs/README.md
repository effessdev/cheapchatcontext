# Developer documentation

## What is CMake

Is it used to make build files. It is a **build system generator**, not a compiler or a build tool itself. Build files generated by CMake are considered generated artifacts and should excluded from version control. CMake uses `CMakeLists.txt` files to define the build configuration.

The build files are generally generated in `/out/build` directory, especially when using Visual Studio. However, Visual Studio often hides the build files in the solution explorer, so you have to use your file explorer to see them.

## After generating build files

We need a build tool to actually build the app using those generated bulid files. The generater that we used to build the build files determines which build tool we need to use.

For example:

| Generator                 | CMake generates          | Build tool |
| ------------------------- | ------------------------ | ---------- |
| **Ninja**                 | `build.ninja`            | `ninja`    |
| **Visual Studio 17 2022** | `.sln`, `.vcxproj`       | MSBuild    |
| **Unix Makefiles**        | `Makefile`               | `make`     |
| **Ninja Multi-Config**    | Multi-config Ninja files | `ninja`    |

You specify the generator with the -G option:

```bash
cmake -G "Ninja" ..
```

or

```bash
cmake -G "Visual Studio 17 2022" -A x64 ..
```

These generate completely different build files from the same CMakeLists.txt. In Visual Studio, when you use `File > Open > Folder`, Visual Studio chooses a default generator for you.
